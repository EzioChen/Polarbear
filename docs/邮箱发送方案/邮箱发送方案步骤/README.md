```
邮箱发送方案步骤/
├── README.md                          # 本文档 - 实施步骤总览
├── 第一步-基础框架搭建.md              # 类型定义、常量、验证器
├── 第二步-配置管理系统.md              # ConfigManager 实现
├── 第三步-邮件发送核心.md              # MailComposer、MailSender、EmailService
```

## 实施阶段概览

### 第一步：基础框架搭建
**目标**：建立项目基础结构，定义类型系统和常量

**核心任务**：
- 目录结构规划与创建
- 依赖引入（Nodemailer、marked）
- 类型定义系统（SMTPConfig、SendEmailOptions、SendResult 等）
- 常量定义（默认值、错误消息、验证规则）
- 基础验证器工具

**前置条件**：无
**预计工期**：1-2 天

---

### 第二步：配置管理系统
**目标**：实现配置文件的加载、验证、存储和更新

**核心任务**：
- ConfigManager 类架构设计
- 配置文件加载功能
- 配置验证功能（必填字段、格式、范围）
- 配置存储与更新（含密码安全存储）
- VS Code 命令集成

**前置条件**：完成第一步
**预计工期**：2-3 天

---

### 第三步：邮件发送核心
**目标**：实现邮件构建、发送和重试机制

**核心任务**：
- MailComposer 邮件构建器
- MailSender 发送执行器（单次连接模式）
- 重试机制（指数退避、可重试错误判断）
- EmailService 核心服务（单例模式）
- 错误处理与日志

**前置条件**：完成第二步
**预计工期**：3-4 天

---

### 第四步：邮件编辑器
**目标**：实现基于 Webview 的可视化邮件编辑器

**核心任务**：
- 邮件编辑器架构设计
- Webview 面板实现
- 前端界面实现（VS Code 主题适配）
- Markdown 编辑与实时预览
- 附件管理
- 邮件发送集成

**前置条件**：完成第三步
**预计工期**：4-5 天

---

### 第五步：UI界面实现
**目标**：实现用户界面和交互功能

**核心任务**：
- 命令注册与菜单集成
- 配置界面实现（向导、模板）
- 状态栏集成（状态显示、快捷菜单）
- 通知与反馈（进度、结果、错误）
- OutputChannel 日志

**前置条件**：完成第三步（可与第四步并行）
**预计工期**：2-3 天

---

### 第六步：集成测试
**目标**：全面测试功能正确性和稳定性

**核心任务**：
- 单元测试（ConfigManager、MailComposer、MailSender、validators）
- 集成测试（模块协作、错误恢复）
- 端到端测试（用户场景）
- 性能测试（耗时、内存）
- 测试文档

**前置条件**：完成第四、五步
**预计工期**：3-4 天

### 关键决策点

#### 1. 单次发送 vs 连接池
**决策**：采用单次发送模式
- 每次发送独立创建连接
- 发送完成后立即关闭
- 简化实现，符合当前需求

#### 2. 密码存储
**决策**：使用 VS Code SecretStorage
- 密码加密存储
- 配置文件不保存明文密码
- 符合安全最佳实践

#### 3. 邮件编辑器技术栈
**决策**：原生 HTML/CSS/JS + marked.js
- 无前端框架，减少体积
- marked.js 处理 Markdown 解析
- 符合 VS Code Webview 最佳实践
