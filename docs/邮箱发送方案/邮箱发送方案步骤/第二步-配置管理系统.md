# 第二步：配置管理系统

## 阶段概述

本阶段实现邮件服务的配置管理功能，包括配置文件的加载、解析、验证、存储和更新。配置管理是整个邮件服务的基础，确保服务能够正确读取和使用用户配置的 SMTP 参数。

---

## 任务一：ConfigManager 类架构设计

### 任务描述

设计 ConfigManager 类的内部结构，明确其职责边界、状态管理和依赖关系。

### 任务目标

1. 定义 ConfigManager 的类结构
2. 明确内部状态管理策略
3. 设计配置缓存机制
4. 规划错误处理策略

### 前置条件

- 已完成第一步的类型定义
- 已了解设计文档中的 ConfigManager 章节

### 详细步骤

#### 1.1 类属性设计

**私有属性：**
- `config`: 当前加载的配置对象（SMTPConfig | null）
- `configPath`: 配置文件路径（string）
- `isLoaded`: 配置是否已加载（boolean）
- `secretStorage`: VS Code SecretStorage 实例（用于安全存储密码）

**属性访问控制：**
- 配置对象通过 getter 方法暴露，防止外部直接修改
- 密码字段通过 SecretStorage 读写，不保存在内存中

#### 1.2 方法职责划分

**核心方法：**
- `loadConfig(path?)`: 从文件加载配置
- `validateConfig(config)`: 验证配置对象
- `getConfig()`: 获取当前配置
- `updateConfig(partial)`: 更新配置
- `saveConfig()`: 保存配置到文件

**辅助方法：**
- `resolveConfigPath(path?)`: 解析配置文件路径
- `parseConfigFile(content)`: 解析配置文件内容
- `mergeWithDefaults(config)`: 与默认值合并
- `maskSensitiveData(config)`: 脱敏敏感数据

#### 1.3 配置缓存策略

**缓存机制：**
- 配置加载后缓存到内存
- 提供 `reload()` 方法强制重新加载
- 文件修改监听（可选，用于自动刷新）

**缓存一致性：**
- 更新配置时同步更新缓存
- 保存失败时回滚缓存

## 任务二：配置文件加载功能

### 任务描述

实现从文件系统加载配置文件的功能，包括路径解析、文件读取和内容解析。

### 任务目标

1. 实现配置文件路径解析
2. 实现文件读取和 JSON 解析
3. 处理文件不存在、格式错误等异常情况
4. 支持相对路径和绝对路径

### 前置条件

- 已完成第一步的常量定义
- 已了解错误码定义

### 详细步骤

#### 2.1 路径解析逻辑

**路径处理规则：**
- 未提供路径时，使用默认路径 `.releasePlan/smtp-config.json`
- 相对路径基于工作区根目录解析
- 绝对路径直接使用
- 路径解析失败时抛出 CONFIG_NOT_FOUND 错误

**路径解析流程：**
1. 获取输入路径或使用默认路径
2. 如果是相对路径，拼接工作区根目录
3. 规范化路径（resolve）
4. 检查文件是否存在
5. 返回绝对路径

#### 2.2 文件读取逻辑

**读取流程：**
1. 使用 fs.promises.readFile 读取文件
2. 指定编码为 'utf-8'
3. 读取失败时抛出 CONFIG_NOT_FOUND 错误

**错误处理：**
- 文件不存在：EmailErrorCode.CONFIG_NOT_FOUND
- 权限不足：EmailErrorCode.CONFIG_INVALID（附带原始错误）
- 其他IO错误：EmailErrorCode.UNKNOWN_ERROR

#### 2.3 内容解析逻辑

**解析流程：**
1. 使用 JSON.parse 解析内容
2. 验证解析结果是否为对象
3. 解析失败时抛出 CONFIG_INVALID 错误

**JSON 错误处理：**
- 捕获 SyntaxError
- 提供详细的错误位置和原因
- 包装为 EmailError 抛出

#### 2.4 与默认值合并

**合并策略：**
- 用户提供配置覆盖默认配置
- 使用深合并（deep merge）
- 数组类型直接替换，不合并

**合并字段：**
- smtp.tls 对象合并
- connection 对象合并
- retry 对象合并
- 其他字段直接赋值

### 预期成果

- 配置文件加载功能完整实现
- 各种异常情况正确处理
- 路径解析符合预期

### 验收标准

- [ ] 路径解析支持相对和绝对路径
- [ ] 文件不存在时抛出正确错误
- [ ] JSON 解析错误时抛出正确错误
- [ ] 配置与默认值正确合并

## 任务三：配置验证功能

### 任务描述

实现配置对象的完整验证逻辑，确保配置符合规范且可用。

### 任务目标

1. 实现必填字段验证
2. 实现字段格式验证
3. 实现字段值范围验证
4. 返回详细的验证结果

### 前置条件

- 已完成第一步的验证器工具
- 已了解 ValidationResult 类型定义

### 详细步骤

#### 3.1 必填字段验证

**必填字段清单：**
- version: 配置版本（string）
- smtp.host: SMTP服务器地址（string）
- smtp.port: SMTP端口（number）
- auth.user: 用户名（string）
- sender.address: 发件人地址（string）

**验证逻辑：**
- 检查字段是否存在（!== undefined && !== null）
- 检查字段类型是否正确
- 缺失字段添加 ValidationError

#### 3.2 字段格式验证

**邮箱格式验证：**
- auth.user 必须是有效邮箱格式
- sender.address 必须是有效邮箱格式
- 使用第一步实现的邮箱验证器

**主机名验证：**
- smtp.host 必须是有效主机名或 IP
- 使用第一步实现的主机名验证器

**端口验证：**
- smtp.port 必须是 1-65535 范围内的整数
- 使用第一步实现的端口验证器

**枚举值验证：**
- auth.type 必须是 'login' | 'plain' | 'oauth2' 之一
- tls.minVersion 必须是支持的 TLS 版本之一

#### 3.3 字段值范围验证

**数值范围验证：**
- connection.timeout: > 0
- connection.greetingTimeout: > 0
- connection.socketTimeout: > 0
- retry.maxRetries: >= 0
- retry.retryDelay: >= 0

**字符串长度验证：**
- smtp.host: 不超过 253 字符
- auth.user: 不超过 254 字符
- sender.name: 不超过 100 字符
- sender.address: 不超过 254 字符

#### 3.4 条件必填验证

**条件逻辑：**
- auth.type 为 'login' 或 'plain' 时，auth.pass 必填
- auth.type 为 'oauth2' 时，auth.accessToken 必填
- secure 为 true 时，建议 port 为 465

#### 3.5 警告检测

**警告场景：**
- 使用非标准端口（非 25/465/587/2525）
- 关闭 TLS 验证（rejectUnauthorized: false）
- 超时时间设置过短（< 5000ms）
- 重试次数过多（> 5）

**警告信息：**
- 提供清晰的警告描述
- 给出改进建议

### 预期成果

- 配置验证功能完整实现
- 验证结果包含错误和警告
- 验证逻辑覆盖所有配置项

## 任务四：配置存储与更新

### 任务描述

实现配置的更新和持久化功能，包括内存更新、文件保存和密码安全存储。

### 任务目标

1. 实现配置更新接口
2. 实现配置文件保存
3. 实现密码安全存储
4. 实现配置重载

### 前置条件

- 已完成配置加载和验证
- 已了解 VS Code SecretStorage API

### 详细步骤

#### 4.1 配置更新接口

**更新策略：**
- 支持部分更新（Partial<SMTPConfig>）
- 更新后自动验证
- 验证失败时回滚更改

**更新流程：**
1. 接收部分配置对象
2. 与当前配置深度合并
3. 验证合并后的配置
4. 验证通过则更新缓存
5. 验证失败则抛出错误

**原子性保证：**
- 先验证后更新
- 验证失败不修改当前配置

#### 4.2 配置文件保存

**保存流程：**
1. 将配置对象转换为 JSON 字符串
2. 格式化 JSON（缩进 2 空格）
3. 写入配置文件
4. 处理写入错误

**敏感数据处理：**
- 保存前移除 auth.pass 字段
- 密码通过 SecretStorage 单独存储
- 保存的 JSON 中 pass 字段设为 null 或移除

**错误处理：**
- 写入失败时抛出错误
- 保留原始配置不变
- 提供详细的错误信息

#### 4.3 密码安全存储

**SecretStorage 使用：**
- 使用 VS Code ExtensionContext.secrets
- 存储键名：'polarbear.smtp.password'
- 存储值：加密后的密码

**密码读写流程：**
- 加载配置时：从 SecretStorage 读取密码并填充到 config
- 保存配置时：将密码存入 SecretStorage，从 JSON 中移除
- 更新配置时：如密码变更则更新 SecretStorage

**安全性考虑：**
- 内存中尽量不长期保存明文密码
- 日志中绝不输出密码
- 配置验证时检查密码是否存在（通过 SecretStorage）

#### 4.4 配置重载

**重载功能：**
- 提供 `reload()` 方法
- 强制从文件重新加载配置
- 清除当前缓存

**重载场景：**
- 用户手动修改配置文件后
- 需要刷新配置时
- 初始化失败重试时

- 配置更新功能完整实现
- 密码安全存储机制建立
- 配置保存和重载功能可用
