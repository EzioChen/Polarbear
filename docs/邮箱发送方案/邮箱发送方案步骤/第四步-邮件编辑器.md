# 第四步：邮件编辑器

## 阶段概述

本阶段实现基于 VS Code Webview 的邮件编辑器，提供可视化的邮件编写界面，支持 Markdown 编辑、实时预览和附件管理功能。

---

## 任务一：邮件编辑器架构设计

### 任务描述

设计邮件编辑器的整体架构，包括 Webview 结构、组件划分和通信机制。

### 任务目标

1. 设计 Webview 面板结构
2. 规划前端组件架构
3. 设计 Extension ↔ Webview 通信协议
4. 确定状态管理策略

### 前置条件

- 已完成邮件发送核心功能
- 已了解 VS Code Webview API

### 详细步骤

#### 1.1 Webview 面板结构

**EmailEditorPanel 类职责：**
- 创建和管理 Webview 面板
- 处理 Extension 与 Webview 的通信
- 管理编辑器生命周期
- 协调邮件发送流程

**面板配置：**
- viewType: 'polarbear.emailEditor'
- title: '邮件编辑器'
- 保留上下文：保持编辑器状态
- 本地资源根目录：允许加载本地资源

#### 1.2 前端架构设计

**页面结构：**
```
邮件编辑器 Webview
├── 头部区域
│   ├── 收件人输入框
│   ├── 抄送输入框
│   ├── 主题输入框
│   └── 工具按钮（发送、草稿、取消）
├── 主体区域
│   ├── 编辑区（左侧/上侧）
│   │   └── Markdown 编辑器
│   └── 预览区（右侧/下侧）
│       └── HTML 预览
└── 底部区域
    ├── 附件列表
    └── 添加附件按钮
```

**技术选型：**
- 纯原生 HTML/CSS/JavaScript（无框架，减少体积）
- Markdown 解析：marked.js
- 代码高亮：highlight.js（可选）

#### 1.3 通信协议设计

**Extension → Webview 消息：**
| 消息类型 | 说明 | 数据 |
|---------|------|------|
| setContent | 设置编辑内容 | { text: string } |
| setAttachments | 设置附件列表 | { attachments: Attachment[] } |
| updatePreview | 更新预览 | { html: string } |
| showError | 显示错误 | { message: string } |
| setConfig | 设置发件人配置 | { sender: SenderConfig } |

**Webview → Extension 消息：**
| 消息类型 | 说明 | 数据 |
|---------|------|------|
| contentChanged | 内容变更 | { text: string } |
| sendEmail | 发送邮件 | { to, cc, subject, text, attachments } |
| saveDraft | 保存草稿 | { draft: DraftData } |
| addAttachment | 添加附件 | - |
| removeAttachment | 移除附件 | { id: string } |
| openConfig | 打开配置 | - |

#### 1.4 状态管理

**状态分类：**
- 编辑器状态：内容、主题、收件人等
- UI 状态：编辑/预览模式、加载状态
- 附件状态：附件列表、上传状态

**状态同步：**
- Webview 内状态由前端管理
- Extension 保存草稿时同步状态
- 发送时从 Webview 获取最新状态

### 详细步骤

#### 2.1 面板创建

**创建流程：**
1. 检查是否已有打开的面板（单例模式）
2. 创建 WebviewPanel 实例
3. 设置 HTML 内容
4. 注册消息处理器
5. 注册关闭事件监听

**单例实现：**
- 静态属性记录当前面板
- 新创建时关闭旧面板
- 或聚焦到已存在面板

#### 2.2 HTML 内容生成

**HTML 结构：**
- 引入 VS Code 主题 CSS 变量
- 引入本地 CSS 文件
- 引入本地 JS 文件
- 设置 CSP（Content Security Policy）

**资源引用：**
- 使用 Webview.asWebviewUri 转换资源路径
- CSS 文件：webview 目录下的样式文件
- JS 文件：webview 目录下的脚本文件
- 图标字体：VS Code 内置图标

#### 2.3 消息通信实现

**Extension → Webview：**
```typescript
webview.postMessage({
  type: 'setContent',
  data: { text: '# Hello' }
});
```

**Webview → Extension：**
```typescript
// Webview 端
const vscode = acquireVsCodeApi();
vscode.postMessage({
  type: 'sendEmail',
  data: { to, subject, text }
});

// Extension 端
webview.onDidReceiveMessage(message => {
  switch (message.type) {
    case 'sendEmail':
      // 处理发送
      break;
  }
});
```

#### 2.4 生命周期管理

**事件处理：**
- onDidDispose：面板关闭时清理资源
- onDidChangeViewState：面板可见性变化
- 保存草稿到 Extension 存储

**资源清理：**
- 关闭时保存未发送的草稿
- 清理临时附件文件
- 释放引用

#### 4.2 实时预览

**预览实现：**
```javascript
const marked = require('marked');

function updatePreview() {
  const markdown = editor.value;
  const html = marked.parse(markdown);
  preview.innerHTML = html;
}

// 防抖处理
let timeout;
editor.addEventListener('input', () => {
  clearTimeout(timeout);
  timeout = setTimeout(updatePreview, 300);
});
```

**预览样式：**
- 基础 Markdown 样式
- 代码块高亮
- 表格样式
- 引用块样式

#### 4.3 工具栏快捷操作

**工具栏按钮：**
| 按钮 | 功能 | 快捷键 |
|------|------|--------|
| B | 粗体 | Ctrl/Cmd + B |
| I | 斜体 | Ctrl/Cmd + I |
| H | 标题 | Ctrl/Cmd + H |
| " | 引用 | Ctrl/Cmd + Shift + Q |
| • | 无序列表 | Ctrl/Cmd + Shift + U |
| 1. | 有序列表 | Ctrl/Cmd + Shift + O |
| []() | 链接 | Ctrl/Cmd + K |
| ![]() | 图片 | Ctrl/Cmd + Shift + I |
| ``` | 代码块 | Ctrl/Cmd + Shift + C |

**实现方式：**
- 在选中文本前后插入标记
- 无选中时插入占位符
- 光标定位到合适位置

#### 4.4 同步滚动

**滚动同步：**
- 编辑区滚动时，预览区同步滚动
- 根据行号计算滚动比例
- 平滑滚动体验

**实现思路：**
```javascript
editor.addEventListener('scroll', () => {
  const ratio = editor.scrollTop / (editor.scrollHeight - editor.clientHeight);
  preview.scrollTop = ratio * (preview.scrollHeight - preview.clientHeight);
});
```

### 前置条件

### 详细步骤

#### 5.1 附件添加

**添加流程：**
1. 用户点击"添加附件"按钮
2. Webview 发送消息到 Extension
3. Extension 打开文件选择对话框
4. 用户选择文件
5. Extension 读取文件并生成临时副本
6. Extension 发送附件信息到 Webview
7. Webview 显示附件

**文件选择：**
```typescript
const uris = await vscode.window.showOpenDialog({
  canSelectFiles: true,
  canSelectFolders: false,
  canSelectMany: true,
  filters: {
    '所有文件': ['*']
  }
});
```

**临时文件处理：**
- 复制文件到临时目录
- 生成唯一文件名
- 记录原始文件名和路径

#### 5.2 附件列表显示

**显示内容：**
- 文件图标（根据类型）
- 文件名
- 文件大小（格式化显示）
- 移除按钮

**样式：**
- 横向排列或纵向列表
- 悬停显示移除按钮
- 文件类型图标

#### 5.3 附件移除

**移除流程：**
1. 用户点击附件上的移除按钮
2. Webview 从列表中移除
3. Webview 通知 Extension
4. Extension 删除临时文件

#### 5.4 大小限制

**限制策略：**
- 单个附件最大 10MB
- 总附件大小最大 25MB
- 超出限制时提示用户

**大小计算：**
```typescript
const totalSize = attachments.reduce((sum, att) => sum + att.size, 0);
if (totalSize > 25 * 1024 * 1024) {
  vscode.window.showWarningMessage('附件总大小超过25MB限制');
}
```

- 抄送（可选）
- 主题
- Markdown 内容

**数据验证：**
- 收件人不能为空
- 主题不能为空
- 内容不能为空
- 邮箱格式验证

#### 6.2 发送流程

**发送步骤：**
1. Webview 发送 sendEmail 消息
2. Extension 收集并验证数据
3. Extension 调用 EmailService.sendEmail
4. 显示发送中状态
5. 接收发送结果
6. 显示发送结果
7. 成功时关闭编辑器

**发送状态：**
- 发送中：显示进度提示
- 成功：显示成功消息，关闭编辑器
- 失败：显示错误消息，保留编辑器

#### 6.3 发送状态反馈

**进度提示：**
```typescript
vscode.window.withProgress({

  cancellable: false
}, async () => {

  return result;
});
```

**结果通知：**
- 成功：showInformationMessage
- 失败：showErrorMessage（带重试按钮）

**草稿恢复：**
