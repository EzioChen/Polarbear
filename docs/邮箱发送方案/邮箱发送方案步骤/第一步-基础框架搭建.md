## 任务三：类型定义系统构建

### 任务描述

根据接口文档，在 `types.ts` 中定义所有 TypeScript 类型和接口，建立完整的类型系统。

### 任务目标

1. 定义配置相关类型（SMTPConfig、RetryConfig等）
2. 定义邮件内容相关类型（SendEmailOptions、Attachment等）
3. 定义结果相关类型（SendResult、ValidationResult等）
4. 定义错误相关类型（EmailError、EmailErrorCode等）

### 前置条件

- 已阅读接口文档中的类型定义章节
- 了解 TypeScript 接口和类型别名

### 详细步骤

#### 3.1 SMTP配置类型定义

**定义 SMTPConfig 接口：**
- version: 配置版本号
- smtp: SMTP服务器配置（host、port、secure、tls）
- auth: 认证配置（type、user、pass、accessToken）
- sender: 发件人配置（name、address）
- connection: 连接参数（timeout、greetingTimeout、socketTimeout）
- retry: 重试配置（enabled、maxRetries、retryDelay、exponentialBackoff）

**类型约束：**
- auth.type 使用联合类型：'login' | 'plain' | 'oauth2'
- port 使用常用端口联合类型或 number
- tls.minVersion 使用联合类型：'TLSv1.0' | 'TLSv1.1' | 'TLSv1.2' | 'TLSv1.3'

#### 3.2 邮件发送选项类型

**定义 SendEmailOptions 接口：**
- to: 收件人（string | string[]）
- cc: 抄送（可选，string | string[]）
- bcc: 密送（可选，string | string[]）
- subject: 主题（必填，string）
- text: 纯文本内容（可选，string）
- html: HTML内容（可选，string）
- attachments: 附件列表（可选，Attachment[]）
- priority: 优先级（可选，'high' | 'normal' | 'low'）
- headers: 自定义头（可选，Record<string, string>）

**约束条件：**
- text 和 html 至少有一个必填
- 收件人邮箱格式需符合 RFC 5322

#### 3.3 附件类型定义

**定义 Attachment 接口：**
- filename: 文件名（必填）
- content: 文件内容（Buffer | string | Readable，可选）
- path: 本地文件路径（可选）
- href: URL地址（可选）
- contentType: MIME类型（可选）
- encoding: 编码方式（可选）
- contentDisposition: 处理方式（'attachment' | 'inline'，可选）
- cid: 内容ID（可选）

**互斥约束：**
- content、path、href 三者至少提供一个

#### 3.4 发送结果类型

**定义 SendResult 接口：**
- success: 发送状态（boolean）
- messageId: 消息ID（可选，string）
- timestamp: 时间戳（string，ISO 8601格式）
- response: SMTP响应（可选，string）
- error: 错误信息（可选，EmailError）
- accepted: 成功收件人列表（string[]）
- rejected: 被拒收件人列表（string[]）
- duration: 发送耗时（number，毫秒）

#### 3.5 错误类型定义

**定义 EmailError 接口：**
- code: 错误码（EmailErrorCode）
- message: 错误描述（string）
- originalError: 原始错误（可选，Error）
- details: 错误详情（可选，Record<string, unknown>）

**定义 EmailErrorCode 枚举：**
按错误类别分组定义：
- 配置错误（CONFIG_NOT_FOUND、CONFIG_INVALID、CONFIG_MISSING_FIELD）
- 连接错误（CONNECTION_FAILED、CONNECTION_TIMEOUT、TLS_ERROR、DNS_ERROR）
- 认证错误（AUTH_FAILED、AUTH_INVALID_CREDENTIALS、AUTH_METHOD_NOT_SUPPORTED）
- 发送错误（SEND_FAILED、SEND_TIMEOUT、RECIPIENT_INVALID、RECIPIENT_REJECTED、MESSAGE_TOO_LARGE、ATTACHMENT_NOT_FOUND）
- 其他错误（UNKNOWN_ERROR、RETRY_EXHAUSTED、SERVICE_NOT_INITIALIZED）

#### 3.6 验证结果类型

**定义 ValidationResult 接口：**
- valid: 验证状态（boolean）
- errors: 错误列表（ValidationError[]）
- warnings: 警告列表（ValidationWarning[]）

**定义 ValidationError 接口：**
- field: 错误字段路径（string）
- message: 错误描述（string）
- code: 错误代码（string）

**定义 ValidationWarning 接口：**
- field: 警告字段路径（string）
- message: 警告描述（string）
- suggestion: 建议（可选，string）

#### 3.7 连接测试结果类型

**定义 ConnectionResult 接口：**
- success: 连接状态（boolean）
- responseTime: 响应时间（可选，number，毫秒）
- response: 服务器响应（可选，string）
- error: 错误信息（可选，EmailError）

### 预期成果

- types.ts 中包含所有必要的类型定义
- 类型定义与接口文档完全一致
- 类型约束和可选/必填属性正确

## 任务五：验证器工具基础

### 任务描述

在 `validators.ts` 中实现基础的验证函数，为后续配置验证和输入验证提供工具支持。

### 任务目标

1. 实现邮箱格式验证
2. 实现主机名验证
3. 实现端口范围验证
4. 实现配置文件结构验证

### 前置条件

- 已完成类型定义
- 已完成常量定义

### 详细步骤

#### 5.1 邮箱验证器

**功能要求：**
- 验证单个邮箱地址格式
- 验证邮箱地址数组
- 返回验证结果（是否有效、错误信息）

**验证规则：**
- 符合邮箱基本格式（local@domain）
- local 部分长度不超过 64 字符
- domain 部分长度不超过 255 字符
- 总长度不超过 254 字符
- 不支持特殊字符（根据 RFC 5322）

#### 5.2 主机名验证器

**功能要求：**
- 验证 SMTP 服务器主机名
- 支持域名和 IP 地址

**验证规则：**
- 域名格式验证
- IPv4 地址格式验证（可选）
- IPv6 地址格式验证（可选）
- 长度限制（不超过 253 字符）

#### 5.3 端口验证器

**功能要求：**
- 验证端口号有效性
- 提供常用端口建议

**验证规则：**
- 端口号范围：1-65535
- 常用 SMTP 端口：25、465、587、2525

#### 5.4 配置文件验证器

**功能要求：**
- 验证配置文件结构完整性
- 验证必填字段存在
- 验证字段值类型和格式
- 返回详细的验证结果

**验证项：**
- version 字段存在且为字符串
- smtp.host 存在且有效
- smtp.port 存在且有效
- auth.user 存在且为有效邮箱
- sender.address 存在且为有效邮箱
- 可选字段类型检查

### 预期成果

- validators.ts 中包含基础验证函数
- 验证函数返回统一的验证结果格式
- 验证逻辑符合设计要求
