# 邮件发送方案设计文档

## 一、方案概述

本文档详细描述了基于 Node.js + Nodemailer 的邮件发送解决方案，为 PolarBear 扩展提供稳定、安全、可配置的邮件发送能力。

## 二、技术选型

### 2.1 核心依赖

| 组件 | 版本 | 用途 |
|------|------|------|
| nodemailer | ^6.9.x | SMTP 邮件发送核心库 |
| @types/nodemailer | ^6.4.x | TypeScript 类型定义 |

### 2.2 技术特性

- **SMTP 协议支持**: 标准 SMTP/ESMTP 协议
- **安全传输**: TLS/SSL 加密连接
- **认证方式**: LOGIN/PLAIN/OAUTH2 等多种认证
- **内容格式**: 纯文本/HTML/混合内容
- **附件支持**: 多附件、流式附件

### 7.2 错误处理策略

| 错误类型 | 重试策略 | 用户提示 | 日志级别 |
|----------|----------|----------|----------|
| 配置错误 | 不重试 | 明确提示配置问题 | error |
| 连接错误 | 指数退避重试 | 提示网络问题 | warn/error |
| 认证错误 | 不重试 | 提示认证失败 | error |
| 发送错误 | 部分重试 | 提示发送失败详情 | warn/error |
| 系统错误 | 不重试 | 提示系统错误 | error |

### 7.3 重试机制

```typescript
// 指数退避算法
function calculateDelay(attempt: number, baseDelay: number): number {
  if (exponentialBackoff) {
    return baseDelay * Math.pow(2, attempt - 1);
  }
  return baseDelay;
}

// 重试流程
async function executeWithRetry<T>(
  operation: () => Promise<T>,
  maxRetries: number,
  retryDelay: number,
  shouldRetry: (error: Error) => boolean
): Promise<T> {
  let lastError: Error;
  
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await operation();
    } catch (error) {
      lastError = error;
      
      if (attempt === maxRetries || !shouldRetry(error)) {
        throw error;
      }
      
      const delay = calculateDelay(attempt, retryDelay);
      await sleep(delay);
    }
  }
  
  throw lastError;
}

## 八、异步处理机制

### 8.1 设计原则

- 所有发送操作均为异步，不阻塞主线程
- 每次发送独立创建连接，发送完成后立即关闭
- 不支持并发发送，单次发送完成后才能进行下一次

### 8.2 简化设计

```typescript
class MailSender {
  private transporter: Transporter | null;
  
  async send(mailOptions: MailOptions): Promise<SendResult> {
    // 创建新连接
    this.transporter = await this.createTransporter();
    try {
      const result = await this.transporter.sendMail(mailOptions);
      return this.formatResult(result);
    } finally {
      // 发送完成后立即关闭连接
      await this.close();
    }
  }
}
```

### 9.5 邮件数据组装

```typescript
interface EmailComposerData {
  to: string[];
  cc?: string[];
  bcc?: string[];
  subject: string;
  markdownContent: string;
  attachments: EmailAttachment[];
}

class EmailComposer {
  // 组装邮件数据
  compose(data: EmailComposerData): SendEmailOptions {
    const html = this.markdownToHtml(data.markdownContent);
    const text = this.markdownToText(data.markdownContent);
    
    return {
      to: data.to,
      cc: data.cc,
      bcc: data.bcc,
      subject: data.subject,
      text: text,
      html: html,
      attachments: data.attachments.map(att => ({
        filename: att.filename,
        path: att.path
      }))
    };
  }
  
  // Markdown 转 HTML
  private markdownToHtml(markdown: string): string {
    return marked(markdown, {
      gfm: true,
      breaks: true,
      headerIds: true
    });
  }
  
  // Markdown 转纯文本 (用于 text 版本)
  private markdownToText(markdown: string): string {
    return markdown
      .replace(/#{1,6}\s/g, '')
      .replace(/\*\*|__/g, '')
      .replace(/\*|_/g, '')
      .replace(/`{1,3}/g, '')
      .replace(/\[([^\]]+)\]\([^)]+\)/g, '$1')
      .replace(/!\[([^\]]*)\]\([^)]+\)/g, '$1');
  }
}
```

## 十一、安全考虑

### 11.1 密码存储

- 配置文件中的密码使用 VS Code SecretStorage 加密存储
- 支持环境变量注入敏感信息
- 提供密码输入命令，避免明文存储

### 11.2 TLS 配置

- 强制 TLS 1.2 以上版本
- 验证服务器证书
- 支持自定义 CA 证书

### 11.3 内容安全

- 验证收件人邮箱格式
- 限制附件大小
- 防止邮件头注入攻击

## 十二、集成方案

### 12.1 与发布流程集成

```typescript
// 发布完成后发送通知
async function onPublishComplete(releaseInfo: ReleaseInfo) {
  const emailService = EmailService.getInstance();
  
  await emailService.sendEmail({
    to: releaseInfo.notifyEmails,
    subject: `【发布通知】${releaseInfo.projectName} ${releaseInfo.version} 已发布`,
    html: generateReleaseEmail(releaseInfo)
  });
}
```

### 12.2 与发布说明集成

```typescript
// 发送发布说明邮件
async function sendReleaseNotes(releaseNotes: ReleaseNotes) {
  const emailService = EmailService.getInstance();
  
  await emailService.sendEmail({
    to: releaseNotes.recipients,
    subject: releaseNotes.title,
    html: renderReleaseNotesToHtml(releaseNotes),
    attachments: releaseNotes.attachments
  });
}
```
